《霓虹清道夫 / Neon Cleaner》互动电影游戏项目周报

本周末项目从“可玩的互动视频原型”推进到“可持续生产 AI 电影内容的流程原型”。当前重点已经明确为先完成电影分镜静帧，再用通过的静帧生成视频并接入互动节点。

查看在线 Demo 查看 GitHub 仓库 查看 Markdown 周报

一、项目定位

Neon Cleaner 是一个互动电影游戏原型。目标不是传统小游戏，也不是单纯 AI 短片，而是让用户先观看电影片段，在关键动作节点接管操作，再根据操作表现进入不同剧情影片。 

观众观看电影片段 -> 关键节点接管操作 -> 系统计算表现 -> 播放不同剧情分支

二、本周核心成果

1 完成第一版可玩的网页互动电影原型
2 验证本地 Sulphur2/LTX-2.3 视频生成环境
3 完成 World Labs / Marble 3D 接管可行性实验
4 建立 Topview 风格的分镜静帧优先制作流程

四、重要流程修正

本周末最重要的判断是：不能一上来直接生成视频。视频模型会同时引入构图、镜头、角色、车辆、场景一致性等不确定因素，容易浪费时间。

现在采用更稳定的流程：

剧本拆分 -> 分镜静帧 -> 人工确认 -> I2V 生成视频 -> 接入互动游戏

六、下周计划

  1. 先完成 4 张关键分镜静帧：旧金山战后大远景、林夏俯瞰城市、敌方装甲车出现、第一人称接管驾驶画面。
  2. 每张图生成 3 个候选版本，筛选通过版。
  3. 用通过静帧进入 Sulphur2/LTX-2.3 I2V，生成 5 到 10 秒视频片段。
  4. 把通过视频替换到网页 demo 中，验证电影到玩法的衔接。
  5. 继续优化第一人称驾驶、碰撞边界、车辆模型、爆炸烟雾和手机端性能。

七、风险和资源需求

  • 直接生成视频风险较高，必须先确认分镜静帧。
  • Marble 3D 世界目前可用于验证空间接管，但还不能作为最终资产。
  • 当前本地视频环境不是静帧生成环境。如需完全本地化分镜图，建议后续单独安装 Flux 或 SDXL。
  • 第一批视觉确认会决定后续视频生成效率，是下周最关键节点。

八、老板需要知道的决策点

  1. 项目已有可玩的网页原型，不只是概念。
  2. 当前核心任务是建立可持续的 AI 电影内容生产流程。
  3. 下周重点不是堆新功能，而是锁定第一批电影级分镜画面。
  4. 如果要提高产出效率，需要稳定的图片生成工具或本地 Flux/SDXL 静帧环境。
  5. 项目下一阶段会以短片段、高质量、可接管为核心，而不是一次生成整部长片。
