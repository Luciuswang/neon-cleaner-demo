## 当前项目定位

`Neon Cleaner` 是一个互动电影游戏原型。目标不是传统小游戏，也不是单纯 AI 短片，而是：

```text
观众先观看电影片段 -> 在关键动作节点接管操作 -> 根据操作结果进入不同剧情影片
```

核心体验方向：

- 用电影级 AI 视频吸引观众。
- 在追车、闪避、Boss 战等节点让玩家“接手剧情”。
- 操作结果决定后续影片内容，形成爽感、代价和遗憾等多结局体验。

### 3. 完成 3D 场景接管实验

我们尝试了 World Labs / Marble 生成的 3D 世界，并导入到了网页实验页。

已验证：

- Marble `.spz` 世界可以加载。
- GLB 碰撞体可以接入。
- 第一人称驾驶视角可以移动。
- 基础碰撞、路径、特效方向可行。

当前问题：

- 生成世界走远后容易出现片状和破碎。
- 车辆仍是低保真占位模型。
- 碰撞和爆炸特效还达不到电影级。
- 手机端运行大型 SPZ 场景压力较大。

结论：3D 接管方向可行，但不应作为当前第一优先级。当前更应该先把电影分镜和视频内容打磨起来。

### 4. 明确 AI 内容生产流程

最重要的流程修正：

```text
剧本拆分 -> 分镜静帧 -> 人工确认 -> I2V 生成视频 -> 接入互动游戏
```

之前直接进入视频生成会浪费时间，因为视频模型会把构图、车辆、镜头方向和场景一致性一次性都变成不确定因素。

现在改为：

1. 先用图片 AI 生成关键分镜静帧。
2. 人工确认画面、镜头、角色、车辆和场景。
3. 再把通过的静帧作为首帧，用 Sulphur2/LTX-2.3 做视频。
4. 最后把视频接入网页互动节点。

## 需要注意的风险

### 1. 直接生成视频风险较高

视频模型很容易出现：

- 车辆变形。
- 镜头不连贯。
- 风格漂移。
- 场景和玩法接不上。

所以必须先做静帧分镜。

### 2. 3D AI 场景还不能当最终资产

Marble 世界可以验证空间接管，但目前真实度、稳定性和手机性能还不够作为最终生产方案。

### 3. 本地还缺静帧生成环境

当前 Sulphur2 环境主要用于视频。若要完全本地化分镜静帧，后续建议单独安装 Flux 或 SDXL，不要和 Sulphur2 视频环境混在一起。

## 需要老板知道的决策点

1. 这个项目已经不是单纯概念，已有可玩的网页原型。
2. 当前最关键的是建立可持续的 AI 电影生产流程。
3. 下周重点不是堆功能，而是把第一批电影分镜画面定下来。
4. 如果要提高效率，需要稳定的图片生成工具或本地 Flux/SDXL 环境。
5. 项目下一阶段会以“短片段、高质量、可接管”为核心，而不是一次生成整部长片。
