# 霓虹清道夫项目周报 - 2026-06-12

## 一、本周目标

本周重点验证“AI 电影短片 + 玩家短时接管”的核心制作路径：先用 AI 生成电影镜头，再在关键动作节点切入可操作的 3D/互动段落。当前 Demo 聚焦 A0 开场段落：战后旧金山、林夏骑摩托追击黑色装甲车队，并在追击高潮处切入玩家接管。

## 二、已完成进展

1. 完成主角方向调整：林夏从原先车内追车设定，改为更有辨识度的未来摩托车手设定，强化动作主角感和年轻用户吸引力。
2. 完成 A0 开场世界观镜头生产实验：使用 Banana 生成 9 张旧金山战后关键帧，并测试 ClipAI / Seedance 多模态视频生成。
3. 确认 Seedance 当前更适合作为开场空镜方案：相比 ClipAI 版本，Seedance 的镜头稳定性、整体氛围和慢节奏更接近项目需要。
4. 完成本地 ComfyUI / Sulphur2 备用流程验证：已生成 A0-S01 城市远景和 A0-S03 装甲车队两个本地视频版本，作为离线备份和 prompt 实验材料。
5. 完成网页 Demo 更新：A0 开场已切换到 Seedance 视频版本，并调整网页遮罩、雨线、扫描线和 UI 面板，使视频更像电影播放而不是网页背景。
6. 完成后续镜头生产包：新增林夏入场、战术电动摩托设定、装甲车队、点火追击/切接管的提示词和制作规范。

## 三、关键验证结论

1. 单个长 prompt 不稳定，必须拆成“镜头级生产”：城市空镜、林夏入场、车队出现、点火追击应分开生成。
2. 多图参考会提高画面统一性，但图太多会导致模型把关键帧当成快速转场，出现烟雾突然生成、车辆凭空出现等问题。
3. 追车镜头必须明确空间关系：林夏在近处后方、车队在远处前方、双方同向远离镜头。只写“追击”不够，模型容易反向理解。
4. 网页端的呈现方式会显著影响观感。强滤镜、UI 遮挡和背景化处理会削弱大片感，需要让关键电影镜头先成为主视觉。
5. 本地模型可作为备用生产线，但当前主镜头的电影感和稳定性更适合用 Seedance / Topview / ClipAI 类工具快速迭代。

## 五、存在问题

1. 目前 A0 开场已有方向，但还没有最终达到“大片开场”的质感，需要继续提高镜头层次、调色和前景构图。
2. 林夏追车镜头出现过方向错误：模型把车队放到林夏身后，导致观感变成敌车追主角。
3. 摩托车仍需正式设计定稿。当前需要先生成未来战术电动摩托设定图，作为后续所有视频镜头和 3D 接管资产的统一参考。
4. 3D 接管段仍是原型级表现，需要继续解决视频到 3D 视角的构图对齐、加载预热和交互爽感。

## 六、下周计划

1. 先完成战术电动摩托设定图，确定车辆外形、灯光语言和可用于游戏接管的轮廓。
2. 生成并筛选 A0-S02 林夏入场镜头，重点保证角色身份稳定、动作克制、与战后旧金山环境统一。
3. 生成 A0-S03 装甲车队进入高架镜头，重点保证车辆出现位置合理、道路透视正确、目标关系清晰。
4. 重新生成 A0-S04 点火追击镜头，明确“林夏追车队”的空间关系，并预留切入 3D 接管的遮挡点。
5. 将选定视频接入网页 Demo，形成更完整的 A0 开场播放链路：城市空镜 -> 林夏入场 -> 车队出现 -> 点火追击 -> 玩家接管。

## 七、本周总结

本周最大的进展是把项目从“单个 AI 视频尝试”推进成了“镜头化生产流程”。我们已经验证了关键帧、Seedance/ClipAI、本地 ComfyUI、网页接管之间的基本链路，也明确了后续制作规则：镜头要拆短、空间关系要写死、视频和网页呈现要一起调。下一步重点不是继续泛泛试模型，而是围绕林夏和摩托建立统一资产，再把追车段落做成可展示的连续样片。
